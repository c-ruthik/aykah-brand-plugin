# Method 5 — Pressure-Test

**Stack:** Six Thinking Hats + Inversion / Reverse Brainstorm + Steelman
**Source:** Edward de Bono (Six Hats), Charlie Munger (Inversion), modern Steelmanning practice

---

## What it does

Stress-tests one specific idea before committing to it. The disciplined version of "what am I missing?"

## When to use

When there is exactly **one** concrete idea or plan and someone needs it pressure-tested:

- "I'm thinking of doing X."
- "Should I do Y?"
- "Help me think this through."

Operates on ONE idea at a time. If there are 5+ candidates, route to **Method 8 — Converge** first.

## Three techniques (run all three in sequence)

### Technique 1 — Six Thinking Hats (de Bono)

Wear each hat in sequence. Commit fully to the hat's perspective before switching. 2–4 substantive bullets per hat.

**[WHITE]** — Facts and data only. What do we actually know? What data is missing? Strip out opinions.

**[RED]** — Gut and emotion, no justification. What does the gut say? Excitement, dread, ambivalence, urgency, suspicion?

**[BLACK]** — Risks and what could go wrong. Worst-case scenario. Where's the weakness? Cost of being wrong about a key assumption?

**[YELLOW]** — Benefits and what could go right. Best-case scenario. Second-order benefits that compound.

**[GREEN]** — Creative alternatives. 3–5 alternative versions of this idea. What if a piece were different? Different approach to the same goal?

**[BLUE]** — Big picture and process. Does this fit the larger strategy? Is now the right time? Is this even the right problem to solve?

Most ideas die at [BLACK]. That's the point.

### Technique 2 — Inversion / Reverse Brainstorm (Munger)

Don't ask "how do we make this succeed?" Ask **"how would we *guarantee* this fails?"**

Generate 6–10 ways to fail catastrophically. Then for each, write the inverse — what would have to be true to prevent that failure mode. The inversion list often becomes the implementation checklist.

### Technique 3 — Steelman

Build the strongest possible case **against** the idea before defending it. Treat each argument-against as if a smart, informed critic with full context were making it.

For each argument-against:

- Mark whether it has merit: `yes` / `no` / `partial`
- Note why
- Note what the rebuttal is, if there is one

The Steelman often surfaces "yes, but only if X" conditions — those conditions are the real decision criteria.

## Output format

```
Idea being tested: <one paragraph>

Six Hats:
  [WHITE] - <2–4 bullets>
  [RED]   - <2–4 bullets>
  [BLACK] - <2–4 bullets>
  [YELLOW]- <2–4 bullets>
  [GREEN] - <3–5 alternatives>
  [BLUE]  - <2–4 bullets>

Inversion:
  Failure mode 1: <how it fails> -> Inverse: <what must be true>
  ...

Steelman (5–8 arguments against):
  - <argument> [merit: yes/no/partial] - rebuttal: ...
  ...

Verdict:
  - Confidence: low / medium / high
  - Conditions that must be true: <bullets>
  - Most important risk to mitigate: ...
  - Recommended next step: <accept / reject / modify / defer>
  - One-line summary: <memorable closing line>
```

## Common failure modes

- Soft Black Hat — if it doesn't sting, run again with more force
- Steelman strawmen — rewrite as a brilliant skeptic
- Skipping Blue Hat because it's uncomfortable
- Producing the verdict before doing the analysis
- Mushy verdicts ("seems good, could work") — use accept / reject / modify / defer only
