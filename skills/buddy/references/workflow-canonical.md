# Canonical Workflows — chaining methods together

Most real brainstorms use 2–4 methods in sequence, not just one. Here are the proven chains.

---

## The full canonical flow

```
Problem / question / decision
       |
       v
   FRAME (Method 1 — 5W + HMW + FP)
       |
       +----+-----+----------+
       |    |     |          |
       v    v     v          v
    DIVERGE  LOTUS    TRIZ
    (Method 2)  (Method 3)  (Method 4)
       |    |     |
       +----+-----+
            |
            v
       CONVERGE (Method 8)
            |
       +----+----+
       |         |
       v         v
   PRESSURE-TEST  SCENARIO
   (Method 5)    (Method 6)
       |         |
       +----+----+
            |
            v
       RED TEAM + PRE-MORTEM (Method 7)
       (final gate before commitment)
            |
            v
        COMMIT
```

## Common chains by use case

### A. Marketing campaign

```
Frame -> Diverge (Crazy 8s) -> Converge -> Pressure-Test
```

1. **Frame** — drill from "do a campaign" to the real underlying business goal
2. **Diverge** — Crazy 8s of campaign angles (forced variety, 2–3 uncomfortable)
3. **Converge** — affinity-group + 2x2 to pick top 2
4. **Pressure-Test** — Six Hats on the chosen angle; surface the conditions for success

### B. Product line / loyalty / content strategy / programmatic SEO

```
Frame -> Lotus Blossom -> Converge -> Pressure-Test
```

1. **Frame** — lock in the specific central theme
2. **Lotus** — 1 + 8 + 64 ideas; surface patterns
3. **Converge** — pick 5–10 carry-forwards, then narrow to 1–3 winners
4. **Pressure-Test** — Six Hats + Inversion on the chosen winner

### C. Tradeoff resolution

```
Frame -> TRIZ -> Pressure-Test
```

1. **Frame** — write the contradiction in canonical form: "We want X without sacrificing Y"
2. **TRIZ** — reality-check, then 5–8 principles, pick the strongest resolution
3. **Pressure-Test** — Six Hats on the resolution; surface the conditions for it to work

### D. Multi-year strategic call

```
Frame -> Lotus Blossom -> Converge -> Scenario Wind-Tunnel -> Red Team
```

1. **Frame** — verb + noun + time horizon
2. **Lotus** — 64-idea map of strategic options
3. **Converge** — narrow to 3 candidates
4. **Scenario** — 4 futures; test each candidate; identify robust moves and hedges
5. **Red Team** — final test on the survivor; key assumptions; kill criteria

### E. Major commitment (build / hire / launch / partnership)

```
Pressure-Test -> Red Team + Pre-Mortem
```

1. **Pressure-Test** — Six Hats + Inversion + Steelman on the proposal
2. **Red Team** — pre-mortem, three adversarial views, key assumptions check
3. Output the cheapest pre-commitment test and kill criteria before commit

## How to chain in practice

When buddy chains methods, output should:

1. **Announce the chain at the start** — "Routing as Frame -> Diverge -> Converge -> Pressure-Test."
2. **Hand off cleanly** — at the end of each method, the output of that method becomes the input of the next.
3. **Track maturity through the chain** — Diverge ideas start at `Raw`, Converge promotes to `Refined`, Pressure-Test promotes to `Ready`.
4. **Run the brand-voice gate at every output checkpoint**, not just at the end.

---

## When NOT to chain

If the user's question fits a single method cleanly, run only that method. Chaining when not needed produces noise.

- "Pressure-test this exact plan" — only Method 5
- "Help me pick from these 12 things" — only Method 8
- "Should we open Calgary in 2027?" — only Method 6 (assuming question is already framed)
