07 · Usage Rules
Do's and Don'ts
Critical rules for anyone working with Aykah colors.
Do Use #B8956A for decorative elements only: star icons, decorative lines, background gradients, icon fills, hover glows.
Never Use #B8956A as text color. It fails WCAG at 2.78:1. Zero exceptions.
Do Use #8B6E4A for any gold-colored text: links, labels, eyebrow categories, price highlights. Passes 4.5:1.
Never Put white text on #B8956A background. Use #7D6040 for buttons instead.
Do Use #7D6040 for buttons with white text. Passes both text and UI contrast requirements.
Never Use #B8B2A8 for readable content. It is exclusively for disabled/inactive UI states.
Do Always pair #E8E3DB borders with a box-shadow or background color change for visibility.
Never Invent new colors outside this system. All hex values must come from this palette.

/* AYKAH COLOR PALETTE — WCAG 2.2 Level AA */ :root { /* Primary — Navy */ --primary: #363B57; --primary-dark: #2A2E45; --primary-light: #4A5070; --primary-tint: #EDEEF2; /* Accent — Gold */ --accent: #B8956A; /* DECORATIVE ONLY */ --accent-text: #8B6E4A; /* gold text, links, labels */ --accent-btn: #7D6040; /* buttons with white text */ --accent-light: #D4BC9A; --accent-glow: rgba(184, 149, 106, 0.08); /* Backgrounds */ --white: #FFFFFF; --ivory: #FAF8F4; --warm: #F7F3EE; --border: #E8E3DB; /* Text */ --text-body: #2C2C2C; --text-muted: #6B6560; --text-light: #76716B; --text-disabled: #B8B2A8; /* DISABLED STATES ONLY */ /* Signals */ --success: #3D7A55; --error: #C0392B; }
